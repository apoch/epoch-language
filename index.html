Epoch Programming Language

Every seasoned programmer knows that building great software involves making some compromises. Choosing a programming language doesn't have to be one of them!

Epoch is a language designed to be opinionated. It has been built from the ground up to emphasize certain mindsets and approaches to building software systems and applications, at all scales. Make no mistake, Epoch is probably not for everyone - but if you're a like-minded developer, working from a similar frame of reference, we hope it scratches your itch as well as it scratches our own.

Where It Is Today

This is a quick snapshot of what the language currently offers:

  • Strong, static typing for safety and correctness
  • Just-in-time compiles to native code for maximum performance
  • Garbage collection for memory management
  • Includes prototype editor with syntax highlighting

Where Things Are Going

There is always room for progress, but these are some of the areas where we feel it is most important for Epoch to grow:

  • Optional garbage collection for both memory and other resources
  • Deterministic resource management for cases where GC is suboptimal or not desired
  • Green thread/fiber model with fast message passing infrastructure for massive parallelism
  • Improved ahead-of-time compiler, including moving to 64-bit platforms
  • More diverse platform support, predominantly Linux, BSD, and OS X
  • Full IDE and first-class tool chain for editing, compiling, and debugging Epoch code

What It Looks Like

To get a taste of what a simple Epoch program looks like, check out the snippet below. More extensive examples can be found in our repositories on GitHub.

//
// Epoch Example Program
// These lines are simply comments.
//

// User-defined function taking one parameter of type string
DisplayMessage : string message
{
    // Invoke the standard library console print function
    // The ; operator concatenates strings together
    print("Important Message: " ; message)
}

// User-defined algebraic sum type (aka discriminated union) 
// This type can hold a string, or it can hold "nothing"
// Epoch does not have null references; instead, "nothing"
// is used to indicate the absence of a value.
type OptionalString : string | nothing

// Overload the DisplayMessage function
// This overload handles a "nothing" value for the message
DisplayMessage : nothing
{
    print("No message provided.")
}

// This function is where the program begins execution
entrypoint :
{
    // Create some variables to hold messages
    OptionalString hello = "Hello world!"
    OptionalString nada = nothing

    // Display our messages to the user
    DisplayMessage(hello)
    DisplayMessage(nada)
}
